# RUNBOOK: AI駆動E2E実行手順

## 目的
- Chrome DevTools MCP と日本語Gherkinで複雑なWebアプリのE2EをAIが自律実行できる構成を用意する。
- 本RUNBOOKを唯一の実行起点とし、前提・手順・成功/失敗条件を一元管理する。
- 通常操作での完走を最優先とし、スニペットはフォールバックとしてのみ使用する。

## 前提条件
- Chrome DevTools MCP が起動済みで、対象アプリのステージング環境にアクセスできる。
- `docs/e2e-testing/guidelines/ai-rules.md` を読み込み、環境・MCP設定・運用ポリシー・自然言語マッピングを適用している。
- 認証情報などの機密値はログ出力しないこと。

## 実行フロー
1. **ガイドライン読込**: `guidelines/ai-rules.md` を読み、`env` と `mcp` の設定、各種ポリシーを適用する。
2. **対象特定**: 利用者の自然言語要望を `mapping` に照らし、対象の `feature@tag` または `scenarios` を選定する。
3. **前処理**: 必要な初期化や状態確認（ログ収集、認証状態の確認など）を整える。
4. **ケース実行**: すべての操作は要素が表示かつ操作可能になるまで `wait_for` する。ロケータは `role/name` → `data-testid` → 安定CSS の順で選択する。
5. **検証**: 重要遷移ごとにスナップショットを取得し、シナリオ完了時はコンソールエラーが0であること、主要API（例: 注文POST）が期待ステータス（例: `201`）を返すことを確認する。
6. **後処理**: 後始末やログ保存が必要な場合は実施し、次の実行に影響を残さない。

## フォールバックポリシー
- 通常操作がリトライ上限まで失敗した場合に限り、対応する `snippets/<name>.js` を **1回だけ** 実行して補完する。
- スニペット実行後は即座に通常操作へ戻り、連続でスニペットに依存しない。
- スニペットはステップで明示された場合（例: `スニペット "fillCardForm" を実行`）のみ即時実行して良い。

## 成功基準
- すべての必須ステップが成功し、定義された検証（UI表示・APIレスポンス・コンソールログなど）が満たされている。
- ステージング環境への操作が完了し、後処理が正常終了している。

## 失敗基準
- 必須ステップまたは検証が失敗・未実施のまま終了した場合。
- フォールバック使用後もシナリオが完了しない場合。
- MCP やネットワークの致命的な障害が解消できなかった場合。

## 代表タグ例
- `@ログイン` : 認証関連のFeature選択。
- `@カート追加` : 商品をカートへ追加する操作。
- `@ペイパル` : ペイパル支払いシナリオを選択。
- `@クレジットカード` : クレジットカード支払いシナリオを選択。
- `@完全購入フロー` : シナリオフォルダ内の長尺シナリオを呼び出す際に使用。

## 参考
- 詳細設定・マッピングは `guidelines/ai-rules.md`。
- 個別の機能テストは `features/`、ユーザージャーニー全体は `scenarios/` を参照。
- フォールバック用コードは `snippets/` に配置。
