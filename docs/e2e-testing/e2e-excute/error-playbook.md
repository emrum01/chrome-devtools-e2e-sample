# エラープレイブック

テスト実行中に遭遇しやすいエラーと、症状・観測方法・対処手順をまとめています。問題発生時は該当項目を参照してください。

## 認証エラー
- **症状**: ログイン状態が確認できない、認証済みページへアクセスできない。
- **観測**: 認証チェックロジックで失敗。ログインページへリダイレクトされる。
- **対処**: テストを即座に終了し、ユーザーに認証を再取得してもらう。必要な場合は [認証情報ポリシー](./credentials.md) を確認。

## 要素が見つからない
- **症状**: 指定した DOM 要素が取得できない。
- **観測**: セレクターで例外が発生する、または空の結果が返る。
- **対処**: 詳細なエラーメッセージを記録し、現在の状態をスクリーンショットで取得する。待機が不十分な場合は [待機戦略](./wait-strategy.md) を適用する。

## ページ読み込み遅延
- **症状**: ページや API の応答が遅い、もしくは完了しない。
- **観測**:
  - `list_network_requests` で長時間続くリクエストを確認。
  - React コンポーネントがレンダリングされていない場合はローディング表示が継続する。
  - `list_console_messages` でブラウザエラーを確認。
- **対処**:
  - タイムアウトする API を特定してログに残す。
  - 必要に応じてスクリーンショットとネットワークログを取得し、開発チームへ共有する。

## タイムアウト
- **症状**: 操作に設定した待機時間を超えても処理が完了しない。
- **観測**: テストフレームワークがタイムアウト例外を返す。
- **対処**: スクリーンショット、ネットワーク状態、コンソールエラーを同時に記録する。待機手順を見直し、必要であれば `wait_for` 条件や再試行回数を調整する。
