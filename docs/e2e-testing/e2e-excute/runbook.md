# E2E 実行ランブック

Chrome DevTools MCP サーバーを使用した E2E テストを円滑に実施するための主要な手順をまとめています。このドキュメント 1 本でテストを開始できます。

## 目的
- MCP サーバーと対象アプリケーションが正常に動作する状態でテストを実行する
- 認証済みのテストアカウントを利用して、独立したテストケースを順次実施する

## 前提条件
1. **MCP サーバー稼働確認**: Chrome DevTools MCP サーバーが起動済みであること。
2. **対象環境の疎通確認**: テスト対象のアプリケーションにアクセスできること。環境 URL は [環境一覧](./environments.md) を参照。
3. **テストアカウントの取得**: 必要な認証情報を準備する。詳細は [認証情報ポリシー](./credentials.md) を参照。

## 実行手順
1. **ブラウザ起動**: テスト対象アプリケーションを必要なタブ構成で開く。
2. **認証状態の確認**: 各タブでログイン状態を即座に確認し、未認証の場合はテストを中断して認証を取得する。
3. **テストケース実行**: 指定された Gherkin テストケースを順番に実施。実行可能なテストは [テストケース索引](./test-cases.md) を参照。
4. **結果検証**: 期待される結果と実際の挙動を比較する。
5. **レポート生成**: 実行結果と観測事項を記録・共有する。

## 実行時のポイント
- 認証・データ整合性・タブ運用に関する詳細ルールは [実行ルール](./rules.md) を参照してください。
- ページ読み込みが遅い場合は [待機戦略](./wait-strategy.md) を適用し、適切に待機・検証します。
- エラーが発生した場合は [エラープレイブック](./error-playbook.md) の手順に沿って調査・対処を行います。

## 重要な注意事項
- 各テストは他のテストに依存せず独立して実行できること。
- テスト実行前後のデータ整合性を維持し、本番環境での実行は避ける。
- テスト完了後は、必要に応じてテストデータのクリーンアップを実施する。
