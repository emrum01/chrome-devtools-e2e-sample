# 実行ルール

E2E テストの品質と再現性を高めるための必須ルールをまとめています。Runbook と併用して遵守してください。

## 認証
- ログインが確認できない場合はテストを即座に終了し、必要な認証手続きを完了する。
- 認証が必要な操作は、必ず認証済みのタブで実行する。

## 独立性
- 各テストケースは他のテスト結果に依存しないよう設計・実行する。
- テスト間で共有するデータは明示的に初期化し、実行順序による副作用を排除する。

## データ整合性
- テスト実行前後でデータの整合性を確認し、必要に応じてクリーンアップを行う。
- 本番環境でのテスト実行は避け、ステージング以下の環境を使用する。

## タブ運用
- ユーザー向けビュー、管理者ビューなど、役割の異なる画面は同一ウィンドウ内の別タブで管理する。
- 操作対象を切り替える際は、対象タブの認証状態と表示内容を再確認する。
- 未認証タブが存在する場合はテストを進行させない。
