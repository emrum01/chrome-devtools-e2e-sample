# Chrome DevTools MCP E2E ドキュメント

Chrome DevTools MCP サーバーを利用したエンドツーエンド (E2E) テスト運用に必要な情報を整理しています。まずは本ページで全体像を把握し、目的に応じて各ドキュメントを参照してください。

## ドキュメント一覧
- [Runbook](./runbook.md): テストを動かすための前提条件と実行フローを網羅したメインガイド。
- [実行ルール](./rules.md): 認証、独立性、データ整合性、タブ運用に関する必須ルール。
- [待機戦略](./wait-strategy.md): ページ読み込みや要素待機の優先順位と確認手順。
- [環境一覧](./environments.md): 利用可能な環境 URL のリストと用途。
- [テストケース索引](./test-cases.md): 利用可能な Gherkin テストケースと前提条件の一覧。
- [エラープレイブック](./error-playbook.md): 想定される失敗パターンと調査・復旧手順。
- [認証情報ポリシー](./credentials.md): テストアカウントの取得方法と取り扱いルール。

## 先に確認しておきたいこと
- テストを実行する場合は、まず [Runbook](./runbook.md) を読み、必要な認証情報がそろっているかを確認してください。
- 待機やエラー対応で迷ったら、対応する専用ドキュメントから優先度と手順を参照します。

## 関連情報
- テストケース選択時は [テストケース索引](./test-cases.md) と [環境一覧](./environments.md) を合わせて参照することで、対象環境と前提条件を素早く特定できます。
