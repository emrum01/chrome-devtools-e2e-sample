# 待機戦略

ページ読み込みや要素の操作可能状態を確認する際の手順と優先順位を示します。遅延が発生した場合は以下の順序で状況を確認してください。

## 1. ネットワーク状態の確認
```
list_network_requests → 進行中の API リクエストがないことを確認
```

## 2. React コンポーネントの読み込み確認
```
evaluate_script → ローディングインジケーターの存在を確認

take_snapshot → スピナーやプログレスバーなどのローディング要素が消失しているか確認
```

## 3. DOM 要素の安定性確認
```
take_snapshot → 対象要素が存在し、操作可能であることを確認
```

## 待機戦略の優先順位
1. **推奨**: ネットワーク状態と React 状態の確認を組み合わせる。
2. **代替**: `wait_for` による特定要素の待機（要素の存在が確実な場合のみ）。
3. **非推奨**: 固定時間の待機。やむを得ない場合のみ使用する。
